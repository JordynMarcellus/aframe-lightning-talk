Focus on virtual reality experiences.

Still good for AR, general 3D interactives

How A-Frame works

HTML5 Custom elements

Allow you to write HTML-like syntax for 3D objects on the web.

Can use stadard DOM selection APIs like document.querySelector (or jQuery!) to manipulate 3D objects

Example (360 video):

<a-scene>
  <a-assets>
    <video id="video-texture" src="assets/test-video.mp4" autoplay muted crossorigin="anonymous" playsinline -webkit-playsinline></video>
  </a-assets>
  <a-videosphere src="#video-texture" rotation="0 255 0"></a-videosphere>
</a-scene>
A-Frame example 360 videosphere

Abstracts three.js

A-Frame's HTML-like syntax makes it a breeze to build

Set attributes to manipulate values like rotation, position and scale easily

Everything in memory

A-Frame keeps its changes in memory to avoid the performance hit of writing to the DOM.

But that means you won't see any changes in the DOM inspector üò¨
